# Release Process

This project is a Chrome extension published through the Chrome Web Store.

## Prerequisites

Install dependencies:

```sh
npm install
```

Create a local `.env` file from `.env.example`:

```env
EXTENSION_ID=
CLIENT_ID=
CLIENT_SECRET=
REFRESH_TOKEN=
```

The `.env` file is ignored by git. Do not commit real Chrome Web Store credentials.

## Credentials

`EXTENSION_ID` comes from the Chrome Web Store Developer Dashboard.

`CLIENT_ID` and `CLIENT_SECRET` come from a Google Cloud OAuth client with access to the Chrome Web Store API.

`REFRESH_TOKEN` is generated by authorising the OAuth client for the Chrome Web Store scope:

```text
https://www.googleapis.com/auth/chromewebstore
```

Treat `CLIENT_SECRET` and `REFRESH_TOKEN` like passwords.

## Build Checks

Use the lower-memory check build during normal development:

```sh
NODE_OPTIONS=--max-old-space-size=1024 npm run build:check
```

This uses production-style webpack configuration but skips minification.

Run the full production build before release:

```sh
NODE_OPTIONS=--max-old-space-size=2048 npm run build
```

The heap cap keeps Node from consuming all system memory. If the build fails from memory pressure, add swap or run it on a larger machine.

## Local Extension Smoke Test

After building, load `dist` as an unpacked extension in Chromium or Chrome.

On this headless machine, use Playwright's bundled Chrome for Testing and `--load-extension=dist` to confirm:

- the extension service worker starts
- `popup/index.html` returns status `200`
- there are no failed popup resource requests
- there are no popup console errors

## Version Bump

Chrome Web Store rejects uploads where `manifest.json` has the same version as the published package.

Bump the patch/build number before upload:

```sh
npm run bump
```

This updates:

- `manifest.json`
- `package.json`
- `package-lock.json`

Then rebuild so `dist/manifest.json` contains the new version:

```sh
NODE_OPTIONS=--max-old-space-size=2048 npm run build
```

## Upload And Publish

Upload only:

```sh
npm run upload
```

Upload and publish:

```sh
npm run upload:publish
```

Use `npm run upload` first when you want to confirm Chrome Web Store accepts the package before publishing.

## Post-Release

After publishing:

1. Commit the source changes, including the version bump.
2. Close the relevant GitHub issue with the commit and published version.
3. Push `master` to GitHub.
4. Delete merged feature branches.
